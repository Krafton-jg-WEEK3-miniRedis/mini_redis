수요 코딩회 (수요일)
목적
•
단 하루, AI를 도구 삼아 팀별 프로젝트에 몰입합니다.

•
결과물 뿐만 아니라 과정도 중요합니다. AI가 작성한 코드를 완벽히 내 것으로 소화해 그 원리를 설명할 수 있어야 합니다.

•
낯선 기능도 Top-down으로 빠르게 풀어내며 '학습 민첩성'을 극대화하고, 현업 레벨의 기술을 직접 부딪쳐보며 실무 감각을 깨우쳐 보세요.

요구사항
•
이번 주 과제는 ‘미니 레디스(Mini Redis)’ 구현입니다.

•
해시 테이블을 활용하여 키-값(key-value) 저장소를 직접 만들어 봅니다.

◦
해당 저장소는 외부에서 사용할 수 있어야 합니다.

•
해시 테이블의 설계 원리, 구현 과정, 주요 함수의 동작 방식을 명확히 설명할 수 있어야 합니다.

•
사용 기술 스택(프레임워크)은 자유롭게 선택할 수 있습니다.

•
“학습”보다는 “구현”이 우선입니다. AI 등을 적극적으로 활용하여 결과물을 만들어내는 것이 최우선 목표입니다.

◦
다만, 만들어진 결과물 중 핵심 로직에 대해서는 이해하고 설명할 수 있어야 합니다.

중점 포인트
•
동시성 문제가 발생하지 않도록 하려면 어떤 아키텍처 또는 구조를 가져야 할지 고민해 봅니다.

•
보관 기간이 만료된 값을 요청받았을 때, 이를 어떻게 처리할지 방안을 마련합니다.

•
외부에서도 Mini Redis를 쉽게 사용할 수 있도록 API 형태의 구조를 설계해 봅니다.

•
사용자가 Redis에 저장된 데이터를 무효화(depreacted)하는 방식에 대해 고민해 봅니다.

•
Redis 서버가 다운되는 상황에서도 보관 중인 데이터를 최대한 안전하게 유지하려면 어떤 방식이 적합할지 검토합니다. (Optional)

품질
•
데이터의 저장, 조회, 삭제 기능이 정상적으로 동작하는지 단위 테스트를 통해 검증합니다.

•
기능 테스트를 통해 외부에서 해당 프로젝트를 사용하는데 문제가 없는지 검증합니다.

◦
간단한 API 생성후 API 응답 값을 미니 레디스(Mini Redis) 에 저장하여 활용합니다.

◦
엣지 케이스를 최대한 고려 합니다.

◦
레디스를 사용할때와 사용하지 않을때의 성능 차이를 비교합니다.
